What is Edge Delivery Services

Edge Delivery Services (EDS) is Adobe's CDN-first architecture for AEM Sites. Pages are pre-rendered as static HTML and cached on Fastly's global CDN edge, achieving sub-50ms Time to First Byte worldwide.

How Three-Phase Loading Works

EDS splits page loading into three phases to maximize Lighthouse scores: Eager (LCP), Lazy (everything else), and Delayed (analytics/martech).

Eager Phase

Only the first section loads in the eager phase. This gets to LCP as fast as possible without loading anything unnecessary.

Lazy Phase

All remaining sections, header, footer, fonts, and lazy styles load after LCP is achieved. Each block's JS and CSS only loads if that block is on the page.

Delayed Phase

Analytics, chat widgets, and martech load 3 seconds after the page is interactive. They never block rendering.

What Lighthouse score does EDS achieve?

EDS achieves a perfect 100 score across all four Lighthouse categories: Performance, Accessibility, Best Practices, and SEO. This is possible because pages are pre-rendered HTML served from CDN with no server processing overhead.

Do I need a JavaScript framework with EDS?

No. EDS uses vanilla JavaScript (ES6+) with no transpiling or build steps. Blocks are self-contained ES modules that are loaded on-demand, only when the block appears on a page. This eliminates framework bundle overhead and hydration costs.

How do authors create content for EDS?

Authors use Universal Editor — a WYSIWYG editor that works directly in the browser, showing a live preview of the page as it will appear to visitors. Content is stored in AEM's JCR repository and rendered to plain HTML at preview/publish time.

What is a Content Fragment in AEM?

A Content Fragment is a piece of structured, schema-driven content stored in AEM. Unlike page content, CFs are reusable across multiple pages and channels. They can be queried via GraphQL API, enabling dynamic blocks that fetch and render data at runtime.
