How to Verify This Experiment

  1. Open DevTools → Application → Cookies → look for hlx-experiment
  2. The cookie value will be hero-cta-test/control OR hero-cta-test/variant-a
  3. The <body> element will have data-experiment="hero-cta-test" and data-variant="control" (or variant-a)
  4. Force a variant for QA: add ?experiment=hero-cta-test&variant=variant-a to the URL
  5. The preview bar at the bottom of the page shows experiment status

How Authors Create an Experiment (No Code Deploy Needed!)

This is the magic of EDS experimentation. Once the plugin is wired in, authors can run new experiments without touching any JavaScript:

  1. Open the page in Universal Editor
  2. Add a Section → in Section Metadata, add a row: Style → experiment-my-test-control
  3. Add another Section → Style → experiment-my-test-variant-a
  4. Set Page Properties: Experiment → my-test
  5. Preview and Publish — the plugin handles traffic split automatically
  6. Check results in RUM Dashboard or Adobe Analytics

What the Plugin Does (vs. the Old Approach)

❌ Old: delayed.js setupExperimentation()

  • Ran 3 seconds after page load
  • Users saw control content flash before variant appeared (FOUC)
  • Random assignment — not sticky across sessions
  • No preview bar for QA
  • No campaign tracking
  • Needed a code deploy for every new experiment

✅ New: @adobe/aem-experimentation Plugin

  • Runs in Eager phase — before decorateMain, zero flash
  • Cookie-based assignment — consistent across sessions
  • Preview bar on *.aem.page for QA team
  • Force variant via URL param: ?variant=variant-a
  • UTM/campaign tracking built in
  • Authors create experiments without code deploys
  • Reports to EDS RUM dashboard automatically
